◆ cu_thread_create()

cu_err_t cu_thread_create ( cu_thread_t *  thr,
void(*)(void *)  func,
void *  arg 
)

The function starts a new thread in calling process, executing by calling func.

A thread must refer to reactor only by locking it's mutex. According to coroutine realisation of concurrency, a thread must not interrupt reactor execution for many time. So, a thread must not use variables concurrently with a some coroutine. It leads to UB.

A thread can be joined, see cu_join.

Parameters
thr- pointer to thread
func- calling function
arg- argument passed to function
Returns
on success, return 0; on error, return error code and content of thr is undefined.
